#### 🔍 代码解释

**示例 2：理解复杂代码**
```
用户：这段代码是做什么的？

function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

AI 助手：这是一个斐波那契数列计算函数。

**功能**：计算第 n 个斐波那契数
**参数**：n - 要计算的位置（从 0 开始）
**返回值**：第 n 个斐波那契数

**工作原理**：
1. 如果 n 小于等于 1，直接返回 n
2. 否则返回前两个斐波那契数的和

**示例**：
- fibonacci(0) = 0
- fibonacci(1) = 1
- fibonacci(5) = 5

**注意**：这个实现对于大数值效率较低，因为它有很多重复计算。
```

#### 🐛 错误调试

**示例 3：解决报错**
```
用户：我的代码报错了，帮我看看

错误信息：Cannot read property 'style' of null

AI 助手：这个错误表示你试图访问一个 null 对象的 style 属性。

**常见原因**：
1. DOM 元素还未加载完成就访问
2. 选择器写错了，找不到元素
3. 元素真的不存在

**解决方案**：
```javascript
// 错误代码
document.getElementById('myDiv').style.color = 'red';

// 正确代码
window.addEventListener('DOMContentLoaded', function() {
  const element = document.getElementById('myDiv');
  if (element) {
    element.style.color = 'red';
  } else {
    console.log('元素未找到');
  }
});
```
```

## 最佳实践

### 1. 如何更好地与 AI 协作

**✅ 好的提问方式：**
- "帮我创建一个带有阴影效果的卡片组件"
- "这段代码在手机上看不正常，怎么修复？"
- "能解释一下这个 CSS 选择器的作用吗？"

**❌ 避免这样提问：**
- "写个代码"
- "这个不对"
- "怎么办？"

### 2. 提问技巧

1. **具体明确**：说清楚你要什么
2. **提供上下文**：给出相关的代码或错误信息
3. **分步骤**：复杂需求分多次提问
4. **迭代优化**：根据 AI 的回答继续提问

## 常见问题

### Q: AI 回答的不准确怎么办？
A:
1. 重新组织你的问题
2. 提供更多上下文
3. 告诉 AI 哪里不对
4. 换个角度提问

### Q: 如何让 AI 理解我的项目结构？
A:
1. 先让 AI 分析项目文件
2. 说明项目使用的技术栈
3. 指出关键文件的作用
