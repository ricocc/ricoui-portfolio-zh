# Claude Code + Kimi 使用指南

## 什么是 Claude Code 和 Kimi？

### Claude Code
Claude Code 是 Anthropic 开发的 AI 编程助手，可以：
- 💻 帮你编写和修改代码
- 🔍 分析代码结构和逻辑
- 🐛 查找和修复错误
- 📚 解释复杂的编程概念
- 🚀 提供开发建议和最佳实践

### Kimi
Kimi 是月之暗面开发的 AI 助手，特别擅长：
- 🌏 中文理解和生成
- 📖 长文本处理
- 💡 创意和构思
- 🔗 多轮对话
- 🎯 任务规划和执行

## 如何配合使用效果最佳

### 1. Claude Code 负责技术，Kimi 负责创意

**Claude Code 专长：**
```
"帮我分析这段代码的性能瓶颈"
"这个错误是什么原因？如何修复？"
"优化这个数据结构"
"解释这个算法的复杂度"
```

**Kimi 专长：**
```
"帮我想一个网站的设计理念"
"这个文案怎么写更吸引人？"
"用户流程怎么设计更合理？"
"帮我规划项目开发步骤"
```

### 2. 协作模式示例

#### 场景：创建个人网站

**第一步：用 Kimi 规划**（创意和结构）
```
"我想做一个个人作品展示网站，
帮我规划一下网站结构和内容，
要有创意但不要太复杂"
```

**第二步：用 Claude Code 实现**（技术实现）
```
"根据这个设计，
用 React 创建一个响应式的作品展示页面，
包含项目卡片、详情模态框和筛选功能"
```

**第三步：用 Kimi 优化**（用户体验）
```
"这个页面的文案和交互流程怎么优化，
让访客更容易理解我的作品？"
```

**第四步：用 Claude Code 完善**（技术细节）
```
"添加加载动画和错误处理，
优化移动端体验，
添加 SEO 优化"
```

## 具体使用技巧

### 与 Claude Code 对话技巧

#### ✅ 好的提问方式
```
"我有一个 React 组件，点击按钮没有反应，
代码如下：[粘贴代码]
帮我找出问题并给出解决方案"
```

```
"我想用 TypeScript 实现一个二叉树，
要求支持插入、删除、查找操作，
请给出完整的实现代码和测试用例"
```

#### ❌ 避免的提问方式
```
"写个代码"  （太模糊）
"这个不对"  （没提供具体信息）
"怎么办？"  （没有上下文）
```

### 与 Kimi 对话技巧

#### ✅ 好的提问方式
```
"我要做一个面向大学生的职业规划网站，
目标用户是大三学生，
请帮我设计网站的：
1. 整体风格和色彩方案
2. 主要功能模块
3. 用户引导流程
4. 内容策略"
```

```
"我在写项目介绍，
这个项目是一个智能简历分析工具，
主要功能是帮助求职者优化简历，
请帮我写一段吸引人的项目介绍，
要专业但不失亲和力"
```

#### ❌ 避免的提问方式
```
"写点东西"  （目标不明确）
"这个不好"  （没说明原因）
"给我灵感"  （太宽泛）
```

## 实战应用场景

### 1. 项目规划和设计

**使用 Kimi：**
```
"我要开发一个在线学习平台，
面向编程初学者，
请帮我：
1. 分析目标用户需求
2. 设计核心功能
3. 规划技术架构
4. 制定开发里程碑"
```

**使用 Claude Code：**
```
"根据这个设计，帮我创建项目的基础结构，
使用 Next.js + TypeScript，
包含用户认证、课程管理、进度跟踪功能"
```

### 2. 代码开发和调试

**使用 Claude Code：**
```
"这段代码运行时报错 'Cannot read property of undefined'，
[粘贴代码]
请帮我：
1. 找出错误原因
2. 提供修复方案
3. 解释如何避免类似错误"
```

**使用 Kimi：**
```
"这个功能的用户体验不够直观，
请帮我设计更好的交互流程，
要考虑到初学者的使用习惯"
```

### 3. 内容创作和优化

**使用 Kimi：**
```
"我需要为技术博客写一篇文章，
主题是 '如何高效学习编程'，
目标读者是编程新手，
请帮我：
1. 设计文章结构
2. 写一段引人入胜的开头
3. 提供具体的建议和案例"
```

**使用 Claude Code：**
```
"帮我创建一个技术博客的 Markdown 模板，
要有代码高亮、目录生成、相关文章推荐功能，
用 React 实现"
```

## 高级协作技巧

### 1. 角色扮演法

**让 Kimi 扮演不同角色：**
```
"你是一个用户体验设计师，
请从专业角度评价这个界面设计：
[描述或截图]
并提出改进建议"
```

**让 Claude Code 扮演技术顾问：**
```
"你是一个资深全栈工程师，
请帮我评估这个技术方案的可行性：
[技术方案描述]
指出潜在问题和优化建议"
```

### 2. 迭代优化法

**第一轮 - 用 Kimi 生成创意：**
```
"我想做一个独特的个人网站，
要有创意，体现我的技术背景和艺术品味，
给我 5 个不同的设计方向"
```

**第二轮 - 用 Claude Code 评估技术可行性：**
```
"从技术上分析，这几个设计方案的实现难度如何？
哪个最适合新手实现？
请给出实现建议"
```

**第三轮 - 用 Kimi 完善设计：**
```
"基于技术评估，我们选择方案 3，
请帮我进一步完善设计细节，
包括：
1. 具体的页面布局
2. 交互细节
3. 内容策略"
```

### 3. 交叉验证法

**先用 Claude Code 生成代码：**
```
"帮我写一个用户注册功能的完整实现，
包括前端表单验证和后端 API"
```

**再用 Kimi 检查用户体验：**
```
"从用户体验角度，检查这个注册流程是否合理？
有没有遗漏重要的用户友好设计？
请给出改进建议"
```

## 常见问题和解决方案

### 问题 1：两个 AI 的建议不一致

**解决方案：**
1. 分析分歧的根本原因
2. 结合两者的优点
3. 根据项目需求做选择
4. 可以询问更具体的问题

**示例：**
```
"Claude Code 建议用 React，Kimi 建议用 Vue，
我的项目是一个中等复杂度的企业官网，
有后台管理功能，团队有 3 个人，
哪个更适合？"
```

### 问题 2：技术实现遇到瓶颈

**处理流程：**
1. 详细描述问题和已尝试的方案
2. 让 Claude Code 提供技术解决方案
3. 让 Kimi 评估是否有替代实现方式
4. 综合考虑选择最佳方案

### 问题 3：设计和开发脱节

**协作方法：**
```
"我按 Kimi 的设计方案开发，
但发现某些设计在技术上实现很复杂，
有什么既保持设计效果又简化开发的方案吗？"
```

## 最佳实践总结

### 1. 明确分工
- **Kimi**：创意、设计、内容、用户体验
- **Claude Code**：技术实现、代码优化、问题解决

### 2. 循序渐进
- 先规划，再实现
- 先整体，再细节
- 先功能，再优化

### 3. 相互验证
- 技术方案让 Kimi 评估用户体验
- 设计方案让 Claude Code 评估可行性
- 重要决策让两个 AI 都给出建议

### 4. 持续迭代
- 根据反馈不断调整
- 结合两个 AI 的优势
- 形成自己的判断能力

## 学习建议

### 新手阶段
1. 多尝试，熟悉两个 AI 的特点
2. 从简单项目开始练习协作
3. 记录有效的提问模式

### 进阶阶段
1. 学会综合运用两个 AI
2. 形成自己的项目开发流程
3. 培养独立思考和判断能力

### 高级阶段
1. 能够指导 AI 更好地协作
2. 创造新的协作模式
3. 成为 AI 协作的专家

记住：**AI 是工具，你是主人**。学会驾驭工具，而不是被工具驾驭！
