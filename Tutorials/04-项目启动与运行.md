# 项目启动与运行

## 获取项目代码

### 方法一：使用 Git 克隆（推荐）

1. 打开 Trae
2. 按 `Ctrl + `` `（反引号）打开终端
3. 克隆项目：
```bash
git clone https://github.com/你的用户名/你的项目.git
cd 你的项目
```

### 方法二：下载 ZIP 包

1. 在 GitHub 项目页面点击 "Code" → "Download ZIP"
2. 解压到本地文件夹
3. 在 Trae 中打开该文件夹

## 安装项目依赖

### 什么是依赖？
依赖就像做菜需要的食材，项目需要这些"食材"才能运行。

### 安装步骤

1. **在 Trae 中打开项目文件夹**
2. **打开终端**（`Ctrl + `` `）
3. **运行安装命令**：
```bash
npm install
```

**等待过程：**
```
📦 Installing packages...
⏳ 这可能需要几分钟...
✅ 看到绿色的 "added xxx packages" 就成功了
```

**如果安装失败：**
```bash
# 尝试清理缓存
npm cache clean --force

# 重新安装
npm install
```

## 启动开发服务器

### 常用启动命令

```bash
# 标准启动命令
npm run dev

# 或者
npm start

# 或者
npm run serve
```

### 成功启动的标志

你会看到类似这样的输出：
```
🚀 Vite v4.0.0  dev server running at:

  > Local:   http://localhost:3000/
  > Network: http://192.168.1.100:3000/

  ready in 523ms
```

### 访问你的网站

1. 打开浏览器
2. 访问 `http://localhost:3000`
3. 看到你的网站啦！🎉

## 项目结构解析

```
我的网站/
├── public/              # 静态文件（图片、图标等）
├── src/                 # 源代码
│   ├── components/      # 组件（按钮、卡片等）
│   ├── pages/          # 页面（首页、关于页等）
│   ├── styles/         # 样式文件（CSS）
│   └── assets/         # 资源文件

```

## 与 AI 协作开发

### 场景 1：项目启动失败

**问题描述给 AI：**
```
"我运行 npm run dev 时报错了，错误信息是：
Error: Cannot find module 'vite'
怎么解决？"
```

**AI 会告诉你：**
```
这个错误说明缺少 'vite' 依赖，解决方法：

1. 删除 node_modules 文件夹
2. 删除 package-lock.json 文件
3. 重新运行 npm install
4. 再运行 npm run dev

具体命令：
rm -rf node_modules package-lock.json
npm install
npm run dev
```

## 常见问题解决

### 问题 1：端口被占用

**错误信息：**
```
Error: Port 3000 is already in use
```

**解决方案：**
```bash
# 换一个端口
npm run dev -- --port 8080

# 或者找到占用程序并关闭
# Windows:
netstat -ano | findstr :3000
taskkill /PID 进程号 /F

# Mac:
lsof -ti:3000 | xargs kill -9
```

### 问题 2：依赖安装失败

**错误信息：**
```
npm ERR! code E404
npm ERR! 404 Not Found
```

**解决方案：**
```bash
# 清理 npm 缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 问题 3：权限错误

**错误信息：**
```
Error: EACCES: permission denied
```

**解决方案：**
```bash
# 修改 npm 默认目录
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'

# 添加到 PATH
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

## 性能优化

### 开发时优化

1. **使用 npm ci（更快）**
```bash
npm ci  # 比 npm install 更快
```

2. **并行安装**
```bash
npm config set maxsockets 10
```

3. **使用国内镜像**
```bash
npm config set registry https://registry.npmmirror.com
```
